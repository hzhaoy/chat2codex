# Codex App-Server Protocol Snapshot

Chat2Codex talks to Codex through `codex app-server --stdio`. The Codex CLI marks this surface as experimental, so this directory keeps a generated schema snapshot for the currently verified local CLI.

Files:

- `manifest.json` records the Codex CLI version, generation command, and schema hash.
- `codex_app_server_protocol.schemas.json` is the bundled JSON Schema generated by the CLI.

Regenerate after upgrading Codex CLI:

```bash
bun run protocol:generate
git diff -- docs/codex-app-server-protocol
```

Run the local protocol smoke test:

```bash
# Fast handshake: initialize + thread/start, no model turn.
bun run smoke:app-server

# Full turn smoke: starts a small model turn and waits for turn/completed.
bun run smoke:app-server:turn

# Approval smoke: induces a command approval, accepts it, and verifies the file write.
bun run smoke:app-server:approval
```

Treat schema diffs as review material before changing `src/agent/codex-runner.ts`. If the smoke test hangs or a method shape changes, fix the runner first and keep chat-facing approval behavior failing closed.
